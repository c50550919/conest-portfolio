## Documentation Standards

### Naming Conventions
- **UPPER_CASE.md** = Permanent reference documentation
- **Session_Notes_YYYY-MM-DD.md** = Temporary session documentation
- **feature_name_STATUS.md** = Status reports (archive when complete)

### Update Guidelines
1. **Update existing docs** instead of creating new versions
2. **Add date stamps** to status reports (e.g., `STATUS_REPORT_2024-11-07.md`)
3. **Create session notes in** `docs/archive/sessions/YYYY-MM/` from the start
4. **Consolidate** if >3 files cover the same topic
5. **Archive** completed session notes after implementation is done

### Documentation Review Schedule
- **Weekly**: Update README.md and status reports
- **Monthly**: Review and consolidate implementation guides
- **Quarterly**: Archive completed session notes
- **Annually**: Full documentation audit
